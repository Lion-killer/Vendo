# ROADMAP / Список можливостей

Інвентар можливостей застосунку TradeRep, зорієнтований на цільову архітектуру, де
бекендом виступатиме база **1С «Управління торгівлею для України»**: HTTP-сервіс,
реалізований **розширенням 1С**, віддає дані прототипу й приймає документ
«Замовлення клієнта» (`ЗаказПокупателя`). Поточний Node-бекенд (`backend/`) —
лише демонстраційний.

Легенда: ✅ реалізовано · 🟡 декоративне (UI є, логіки немає) · 🔲 заплановано

> при правках. Номери стабільні; нові пункти додаються в кінець відповідного розділу.

## Сутності в 1С: що вже є, чого бракує

Аналіз вивантаження метаданих (`TradeUkr/`, **не** в git) показав, що типова УТ вже
містить майже все потрібне — створювати треба лише довідник «Мобільні пристрої».

| Потреба застосунку | Об'єкт у базі 1С | Стан |
|---|---|---|
| Товари (багаторівневі) | `Справочник.Номенклатура` + `НоменклатурныеГруппы` | ✅ є |
| Залишки по складу | `РегистрНакопления.ТоварыНаСкладах` (− `ТоварыВРезервеНаСкладах`) | ✅ є |
| Ціни | `РегистрСведений.ЦеныНоменклатуры`, `Справочник.ТипыЦенНоменклатуры` | ✅ є |
| Контрагенти | `Справочник.Контрагенты` (+ `КонтактныеЛицаКонтрагентов`) | ✅ є |
| Борг | `РегистрНакопления.ВзаиморасчетыСКонтрагентами` | ✅ є |
| Менеджер | `Справочник.Пользователи` | ✅ є |
| Склади | `Справочник.Склады` | ✅ є |
| Документ замовлення | `Документ.ЗаказПокупателя` | ✅ є |
| Оплата на місці | `Документ.ПриходныйКассовыйОрдер` | ✅ є |
| **Прив'язка пристрою → склад/ціна/менеджер/контрагенти/товари** | **немає** | 🔲 створити: `Справочник.МобильныеУстройства` (розширенням) |

> **Тип ціни** окремою сутністю в застосунку не потрібен (п. 36 старої версії): він
> мапиться через менеджера/мобільний пристрій (п. 29) і застосовується на бекенді
> при віддачі цін.

## Пропозиції візуалізації багаторівневого каталогу (п. 31)

`Номенклатура` в УТ — ієрархічний довідник (групи `НоменклатурныеГруппы` довільної
вкладеності). Поточний акордеон (один рівень) для цього замалий. Варіанти:

1. **Drill-down (рекомендовано для мобільного)** — тап по групі відкриває її вміст
   на весь екран, «хлібні крихти» зверху для повернення. Звично для мобільних,
   добре працює на глибокій вкладеності.
2. **Розкривне дерево (поточний акордеон, розширений)** — вкладені групи з відступами.
   Просто, але незручно при глибині > 2 рівнів на малому екрані.
3. **Гібрид** — drill-down по групах + наскрізний пошук (п. 5), що ігнорує ієрархію
   й показує плаский результат із зазначенням групи.

Рекомендація: **варіант 1 (drill-down) + наскрізний пошук** — це стандарт для
мобільних B2B-каталогів і найкраще лягає на ієрархію 1С.

## Підсумок

Прототип закриває **клієнтську вітрину торгпреда** — каталог, контрагенти, кошик-замовлення,
список замовлень — і має сильну **offline-first** основу.

Перехід на 1С УТ — це насамперед **HTTP-сервіс розширенням 1С** і новий довідник
**«Мобільні пристрої»**, навколо якого будується вся фільтрація даних (склад, ціна,
менеджер, контрагенти, товари). Решта сутностей у базі вже є.

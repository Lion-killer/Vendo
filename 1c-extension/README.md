# Розширення 1С — бекенд TradeRep

Розширення конфігурації (CFE) для **1С «Управління торгівлею для України»**, яке
виступає реальним бекендом мобільного застосунку TradeRep замість демонстраційного
Node-сервера (`../backend/`).

- Призначення: **AddOn** (доповнення)
- Префікс об'єктів: `трепр_`
- Сумісність: `8.3.19` (як задає база для розширень)
- Цільова база: УТ для України 2.3 (сумісність 8.2.13)

Реалізує issues [#5](https://github.com/Lion-killer/TradeRep/issues/5) (довідник «Мобільні пристрої») та [#2](https://github.com/Lion-killer/TradeRep/issues/2) (HTTP-сервіс).

## Склад

```
src/
├─ Configuration.xml                       # маніфест розширення
├─ Catalogs/трепр_МобильныеУстройства.xml   # довідник «Мобільні пристрої»
├─ HTTPServices/трепр_МобильноеПриложение/  # HTTP-сервіс (REST API)
│  └─ Ext/Module.bsl                        # обробники методів
├─ Roles/трепр_ОсновнаяРоль.xml
└─ Languages/Русский.xml
```

## Довідник «Мобільні пристрої» (`трепр_МобильныеУстройства`)

Центральна сутність: один запис = один планшет/користувач застосунку. До нього
прив'язано все, що бачить торгпред.

| Реквізит | Тип | Призначення |
|---|---|---|
| `ИдентификаторУстройства` | Строка(36), індекс | GUID із QR-коду для автентифікації |
| `Склад` | СправочникСсылка.Склады | джерело залишків |
| `ТипЦен` | СправочникСсылка.ТипыЦенНоменклатуры | за яким типом цін віддавати ціни |
| `Менеджер` | СправочникСсылка.Пользователи | відповідальний у замовленнях |
| ТЧ `Контрагенты` | Контрагент | доступні клієнти |
| ТЧ `Товары` | Номенклатура | доступна номенклатура |

## HTTP-сервіс (`трепр_МобильноеПриложение`, rootURL `traderep`)

Контракт **1:1** з демо-бекендом — фронтенд не змінюється (лише адреса API).
Пристрій передається заголовком `X-Device-Id` або параметром `?device=<GUID>`.

| Метод | Шлях | Опис |
|---|---|---|
| POST | `/auth` | `{deviceId}` → валідація пристрою, користувач + токен |
| GET | `/products` | номенклатура пристрою з ціною (тип цін) і залишком (склад − резерв) |
| GET | `/categories` | групи номенклатури пристрою |
| GET | `/customers` | контрагенти пристрою з боргом (`ВзаиморасчетыСКонтрагентами`) |
| GET | `/orders?startDate=&endDate=` | замовлення менеджера за період |
| POST | `/orders` | створити `ЗаказПокупателя` |
| PUT | `/orders/{num}` | оновити склад/контрагента |
| DELETE | `/orders/{num}` | видалити замовлення |

`id` товарів/контрагентів у API = рядковий GUID посилання 1С. Ціна позиції
фіксується (snapshot) у `Товары.Цена`, як у демо-бекенді.

## Як підключити

1. Завантажити XML-джерела в розширення бази (Конфігуратор / `LoadConfigFromFiles`
   у режим розширення, або через утиліту).
2. Опублікувати HTTP-сервіс на веб-сервері (Apache/IIS) — отримати URL виду
   `https://<сервер>/<база>/hs/traderep`.
3. Створити записи довідника «Мобільні пристрої», задати склад/тип цін/менеджера,
   заповнити контрагентів і товари, згенерувати `ИдентификаторУстройства`.
4. У застосунку прописати цей URL і GUID (через QR-код — issues #1/#27/#28).

## Що потребує вивірки на боевій базі

> Код створення `ЗаказПокупателя` використовує реквізити, специфічні для конкретної
> конфігурації. Перед бойовим використанням перевірити (позначено `// TODO` в модулі):
> - **Организация** — зараз береться перша з довідника; задати правило вибору.
> - **ДоговорКонтрагента** — береться `ОсновнойДоговорВзаиморасчетов`; може бути обов'язковим.
> - **Валюта/НДС** — за потреби заповнити явно.
> - **Проведення** документа (зараз лише `Записать()` без проведення).
